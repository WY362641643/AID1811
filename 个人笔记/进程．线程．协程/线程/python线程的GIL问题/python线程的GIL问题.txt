
python线程的GIL问题（全局解释器锁）

GIL ： 由于python解释器中加入了全局解释器锁，导致
    python解释在同一时刻只能解释一个线程，所以大大
    降低了python多线程的执行效率。

后果： python线程一般会用在大量IO阻塞程序或者高延迟
    的程序中，比如网络编程。因为python线程遇到阻塞时
    会主动让出解释器。

GIL解决建议：
     * 尽量使用进程完成并发
     * 修改c解释器或者不使用c解释器  java c#
     * 可以使用多种组合并发
